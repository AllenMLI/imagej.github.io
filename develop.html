Tools & Features

  • Reproducibility. Use of Git to manage code, and Maven for version management facilitates scientific reproducibility. Tools like script templates allow you to easily distribute working examples of your tools.
  • Open source. ImageJ is a tool for science. It is necessarily developed for and by the community; thus the core must be open. That said, within the ImageJ ecosystem there are opportunities for commercialization - as with the immensely successful Bio-Formats project.
  • Extensibility. Built on the powerful SciJava plugin framework, practically any aspect of ImageJ can be customized (even the UI). Popular plugin types include Ops, image formats, and script parameters.

Join the Community

  • The mailing lists will connect you with thousands of other ImageJ users and developers. Ask and answer questions here and become part of the community that has driven ImageJ's success.
  • In addition to hosting the ImageJ source, we use GitHub for issue tracking and pull requests. Contributions are always welcome! For best practices, see the code style guide.
  • You can also submit a complete plugin repository to become part of the ImageJ core or Fiji distribution. In that case, please see our contribution requirements.
